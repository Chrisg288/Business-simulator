# Master Interface Concept v2.2

## Main correction

The TreeView must not become empty or merely decorative. It is the classification/navigation source. When the user selects a TreeView node, the center workspace must update based on the selected Activity Tool.

## Activity Tool presentation rule

```text
Selected TreeView Node × Activity Tool = Center Presentation
```

- Swimlane: show stable sector swimlanes and lane objects.
- Datagrid: show typed rows/records for the selected classification.
- Model: show a graph/model workspace tied to the selected node.
- Simulate: show scenario setup, agents, controls, outputs, and failure points.
- Assess: show evidence/credit/value measurement.
- Compare / Transact: show comparable offer rows and transaction actions.
- Objects / Definitions: show schema/field/presentation rules for the selected context.

## Home rule

Home is not another sector. Home is the orientation layer. It opens with a clickable five-sector economy wheel. Clicking a wedge drills into the matching sector.

## Compact status stream rule

VRQs, messages, evidence records, and status items should each occupy one line of text. Icons or badges must be no taller than the text row.
